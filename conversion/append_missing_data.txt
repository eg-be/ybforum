Am einfachsten machen wir das Ganz auf ZAME
und verändern nur die post_table und thread_table
=================================================

Neuen Dump vorbereiten:

 gunzip 20180108_1898admin.sql.gz

Und diesen Dump in die alte DB einspielen:

  mysql -p -u ybforum old1898 < 20180108_1898admin.sql

Feststellen ab wo neu importiert werden soll.

Ein Startupunkt ist z.Bsp. der Thread mit dem TitelOT: Zahnarzt:
 idpost: 669685
 ithread: 78589
 old_no: 1080532
 
 
Ab dort alle Daten löschen:

 DELETE FROM post_table WHERE idpost >= 669685
 DELETE FROM thread_table WHERE idthread >= 78589

Und die Auto-Inc Werte zurück setzen:

 ALTER TABLE post_table AUTO_INCREMENT = 669685
 ALTER TABLE thread_table AUTO_INCREMENT = 78589
 
Jetzt den Importer neu laufen lassen, aber erst ab dem Post mit 
old_no 1080532. Dazu den Code entsprechend editieren, in ImportThreads,
im entsprechenden SQL-Statement.

EnsureUserId im Importer verändern, so dass eine Exception geworfen wird wenn
ein User nicht existiert.

Vor dem laufen lassen noch die Parameter in DataImporter.php anpassen.

 php -d zend.assertions=1 ImportThreads.php


Jetzt einen neuen Dump erstellen, aber nur die Tabllen
post_table und thread_table dumpen:

 mysqldump -u ybforum -p ybforum thread_table post_table > 20180109_ybforum_thread_post.dump.sql

Zip für den Transfer erstellen:

 gzip 20180109_ybforum_thread_post.dump.sql
 
Transferieren auf die neue Umgebung..

Mit SSH auf die Umgebung bei hostfactory verbinden.
Dump extrahieren:

 gunzip 20180109_ybforum_thread_post.dump.sql.gz

Verbindung mit MySQL herstellen und alte Tabellen thread und post mit Daten aus DUMP überschreiben.
Vorher die Seite auf read-only setzen (am einfachsten den Benutzer in model/DbConfig.php auf
etwas ungültiges setzen)

  mysql --host=server09.hostfactory.ch --port=3306 -u ybforum_usr -pgE9YMa%AJUna ybforum < 20180109_ybforum_thread_post.dump.sql
  
Und die DB wieder aktivieren.
